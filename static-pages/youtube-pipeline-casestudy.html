פרק 1 -המטרה

תהליך ידני חוזר הופך למערכת

נקודת המוצא האישית פשוטה: סיכום סרטונים תוך כדי צפייה גוזל קשב וזמן. הטיפול בה -תהליך תפעולי לכל דבר

הבעיה התפעולית

זיהיתי אצלי תהליך ידני שחוזר על עצמו: צפייה בסרטון, עצירות לרישום הערות, ותוצר שיוצא שונה בכל פעם. במקום להתייעל בתוך התהליך -הוצאתי אותו לאוטומציה מלאה, כמו שמטפלים בכל תהליך חוזר בארגון: אפיון, בחירת כלים לפי עלות מול ערך, הטמעה ומדידה. היום אני מוסיפה URL אחד -והמערכת עושה את כל השאר.

פרק 2 -עלויות

כמה זה עולה?

המערכת כולה פועלת על שכבות חינמיות -רק Claude API עולה כסף, ולפי צריכה בלבד: חודש ללא סרטונים = ₪0. אין שום רכיב קבוע

כלי תפקיד תוכנית עלות
Google Sheets בסיס נתונים + סטטוס Free ₪0
Google Apps Script Orchestration -מריץ את כל הלוגיקה Free (6 דק' ריצה/יום) ₪0
Supadata.ai חילוץ תמלול מיוטיוב Free (100 סרטונים/חודש) ₪0
Claude API יצירת HTML מהתמלול Pay per use ~₪0.55 (~$0.15) / סרטון
Netlify אחסון + פרסום האתר Free (300 deploys/חודש) ₪0
Gmail שליחת התראות Free ₪0
סך הכל לסרטון ~₪0.55 לסרטון · ~₪22 בחודש בקצב הנוכחי (40 סרטונים)

* מחושב על ~4K tokens קלט + ~9K tokens פלט עם claude-sonnet-4-6. ריצות פיתוח ו-debugging עולות יותר -בשימוש שוטף העלות נמוכה משמעותית.

פרק 3 -החלטות ארכיטקטורה

מה נדחה -ולמה

ארכיטקטורה נמדדת גם במה שלא נבנה. שלוש ההחלטות שקבעו את המבנה -עם המספרים

אלטרנטיבה שנשקלה למה נפסלה מה נבחר במקום
Make.com / n8n בענן ₪34-80 בחודש, לתמיד -עלות קבועה על תהליך שרץ דקות בודדות ביום Apps Script -₪0, רץ על שרתי Google
שרת ייעודי (VPS) עדכונים, אבטחה וזמינות -overhead תפעולי קבוע על מערכת של משתמשת אחת Serverless מלא -Apps Script + Netlify
גישה ישירה לכתוביות YouTube YouTube חוסם בקשות תמלול משרתי Google -נשבר כבר בבדיקה הראשונה Supadata.ai -100 סרטונים בחודש בחינם

העיקרון המנחה: עלות קבועה מוצדקת רק כשיש עומס קבוע. בעומס של דקות ביום -משלמים רק על מה שרץ בפועל. זו אותה שאלה שנכון לשאול על כל כלי בארגון: כמה הוא עולה בשנה, מול מה שהוא באמת עושה.

בקרת איכות בפרודקשן

Code Review אחרי העלייה לאוויר -שלושה תיקונים

מערכת חיה היא מערכת שמתחזקים. ביקורת קוד על המערכת הרצה איתרה שלושה ליקויים -תוקנו ופורסמו באותו יום (v2026-06-30)

אבטחה: endpoint המחיקה היה פתוח -כל מי שהחזיק ב-URL יכול היה למחוק כל סרטון מהספרייה
כל פעולת מחיקה דורשת כעת קוד אדמין סודי -ניהול רק למי שמחזיק בו
מייל כפול נשלח לבעלת המערכת על כל סרטון חדש -כתובת הבעלים הופיעה גם ברשימת המנויים
תוקן שורש הבעיה (סקריפט בדיקה שנשאר פעיל) + סינון כפילויות בשליחה עצמה
קוד JavaScript מת זרק שגיאה שקטה בכל טעינה של עמוד הספרייה
הוסר -הקונסול נקי. שגיאה שקטה היום היא באג מסתורי בעוד חצי שנה

שלושת התיקונים מתועדים בפומבי ב-GitHub -כולל מה גרם לכל באג ולמה נבחר דווקא התיקון הזה.

גבולות ידועים ותוכנית המשך

מה נשבר בסקייל -ומה התוכנית

המערכת תוכננה במודע לעומס הנוכחי (~40 סרטונים בחודש). הגבולות מוכרים, מתועדים -ויש להם תוכנית

1
Apps Script: מוגבל ל-6 דקות לריצה -מספיק לסרטון בודד, לא לעיבוד אצווה גדול
2
Supadata: 100 סרטונים בחודש בחינם -מעבר לכך נדרשת תוכנית בתשלום
3
אחסון ב-ScriptProperties: פתרון נכון לעשרות עמודים -לא לאלפים. מעבר לכך: בסיס נתונים אמיתי
Roadmap -בשלבים, כל שלב עולה לאוויר לפני שמתחילים את הבא

שלב 2: גרסת Multi-tenant -הרשמה עם Supabase Auth, כל משתמש מביא מפתח Claude משלו (BYOK). יעד: 3-5 משתמשים ראשונים. שלב 3: עיצובים לבחירה + subdomain אישי -השקה פומבית. בזכות מודל BYOK, עלות התפעול נשארת ~₪0 גם בגדילה.

שאר 6 הצעדים מתרחשים אוטומטית מיידית.
1
אני מוסיפה URL + שם הסרטון ל-Google Sheet

זה הדבר היחיד שאני עושה ידנית. הנושא אופציונלי -אם לא מוסיפה, המערכת מסיקה לבד.

2
Trigger אוטומטי עם שינוי בשיט -אוטומטי

Google Apps Script מופעל ברגע שמוסיפים URL חדש לשיט. אין המתנה - הסקריפט מתחיל לעבד מיד.

ScriptApp.newTrigger("processNewVideos").forSpreadsheet(SpreadsheetApp.getActiveSpreadsheet()).onChange().create()
3
חילוץ תמלול -Supadata API -אוטומטי

הסקריפט שולח את ה-URL ל-Supadata.ai ומקבל תמלול טקסטואלי מלא של הסרטון.

GET api.supadata.ai/v1/youtube/transcript?url=...
4
יצירת סיכום HTML -Claude API -אוטומטי

התמלול + Prompt עם CSS מלא + הוראות מבנה - Claude מחזיר דף HTML מעוצב בעברית RTL.

POST api.anthropic.com/v1/messages · claude-sonnet-4-6 · max_tokens: 16000
הפרומפט המלא ששלחתי ל-Claude לחצי לפתיחה 
You generate a complete RTL Hebrew HTML summary page from a YouTube transcript.

IRON RULE: ONLY content from the transcript. Zero inventions. Zero additions.
If something is NOT explicitly said in the transcript - do NOT write it. Not as an example, not as a tip, not as an explanation. If the transcript has only 2 tools, write 2 - not 3. If a name or concept is not mentioned - do not add it. Every sentence must be traceable to the transcript.

Title: [כותרת הסרטון]
Source: [URL של הסרטון]

[כאן מוכנסת חבילת CSS מלאה - ראי קוד Apps Script]

Build this EXACT HTML structure:

1. <a href='/' class='back'>חזרה לכל הסרטונים</a>

2. .hero: .tag (speaker+date), h1 (Hebrew title), .subtitle (one sentence),
   <a class='yt' href='SOURCE_URL'>▶ לסרטון המקורי</a>

3. .central-idea: core message of the video

4. .level (רמה 1): .level-badge 'רמה 1', h2, .tagline,
   .tool-grid with .tool-card items (each tool: .tool-name + p description),
   .actions-box with ol>li actionable steps

5. .level (רמה 2): .level-badge 'רמה 2', h2, .tagline,
   .context-ways with .way items (.way-num + p for each approach),
   .actions-box

6. .level (רמה 3): .level-badge 'רמה 3', h2, .tagline,
   .definition-box (exact quote translated to Hebrew + original in parentheses),
   .context-ways, .example-box, .system-grid, .actions-box

7. .final-message: closing thought + .source with links

8. <script> IntersectionObserver adds class 'anim-fadeup' to '.anim-ready' elements

NO dark backgrounds except .definition-box.
Mixed Hebrew/English titles: add dir='rtl' to h1, wrap English in <bdi> tags.
Quotes in .definition-box: translate to Hebrew, original English in parentheses.
Return complete HTML only. No markdown fences.

TRANSCRIPT:
[התמלול המלא - עד 12,000 תווים]
